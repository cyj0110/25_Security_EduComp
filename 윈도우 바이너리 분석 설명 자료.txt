[x32dbg]
동적 분석 도구: 프로그램을 실행시키면서 분석할 수 있는 도구 

[인터페이스]
CPU 영역: 현재 실행중인 명령어들. 그리고 그 명령어의 메모리 주소와 실제 기계어를 확인할 수 있음.
Register 영역: EAX, EIP, Flags 같은 레지스터들의 변화 상태를 알 수 있음.
Dump 영역: 주소에 들어있는 값들을 실제 16진수 값과 ASCII코드로 알 수 있음.
Stack 영역: 스택 영역에 있는 데이터들을 확인할 수 있음.

[단축키]
F9: breakpoint까지 진행 (breakpoint는 중단점으로, 프로그램을 실행하면서 특정 명령어에 중단점을 걸어 해당 지점까지 F9로 실행시킬 수 있음.) 
F8: 함수 진입은 건너뛰고 한 루틴씩 진행 (call 함수 직접 보여주면서 테스트)
F7: 함수 진입 후 루틴 진행 (call 함수 직접 보여주면서 테스트)
F2: breakpoint 설정/해제
Ctrl + F2: 프로그램 재시작

cmp: 인자 값을 비교하는 명령어
je: jump if equal - 앞의 명령어 결과에서 같으면 점프, 같지 않으면 점프 X
